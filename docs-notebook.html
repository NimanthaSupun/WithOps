Chapter I

Getting Started

Everything you need to know to set up your first project and write your first line of code.

5 min read Beginner Setup

Welcome to Folio. This guide walks you through the fundamentals — from installing the package to making your first API call. By the end of this page, you'll have a working project ready to build on.

📌 Before you begin

Ensure you have Node.js 18+ and a valid API key from your dashboard. You can generate keys under Settings → Developer → API Keys.

Prerequisites

Folio works in any modern JavaScript or TypeScript environment. You'll need the following before proceeding:

1

Install Node.js

Download Node.js 18 or later from nodejs.org. We recommend using the LTS release for stability.

2

Create an Account

Sign up at folio.dev and navigate to your dashboard. A free tier is available — no credit card required.

3

Obtain your API Key

Under Settings → Developer, generate your first secret key. Store it securely — it will only be shown once.

Installation

Install the Folio SDK using your preferred package manager. The package is available on the public npm registry.

bash 
# Using npm
npm install @folio/sdk

# Or with yarn
yarn add @folio/sdk

# Or with pnpm
pnpm add @folio/sdk

Your First API Call

Once installed, initialize the client with your API key. We recommend storing your key in an environment variable rather than hardcoding it in your source.

typescript 
import { Folio } from '@folio/sdk'

// Initialize with your API key
const client = new Folio({
  apiKey: process.env.FOLIO_API_KEY,
  environment: 'production',
})

// Make your first request
const response = await client.documents.list({
  limit: 10,
  sort: 'created_at',
})

console.log(response.data)

Understanding the Response

Every API response follows a consistent envelope structure. The data field contains your requested resource, while meta provides pagination details. Errors are surfaced with a clear error.code and a human-readable error.message.

If the call succeeds, you'll receive a 200 OK along with your document list. From here you're ready to explore the rest of the API surface — routing, authentication, webhooks, and more.
