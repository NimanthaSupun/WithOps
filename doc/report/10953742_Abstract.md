# Abstract

Modern software development relies heavily on Continuous Integration and Continuous Deployment pipelines, yet over 60% of these pipelines contain critical security misconfigurations that remain undetected until exploited. Organisations simultaneously lack automated tools to assess their DevSecOps maturity against established frameworks such as the OWASP DevSecOps Maturity Model, and threat modelling practices remain disconnected from development workflows, leading 67% of teams to skip them entirely. This project aimed to design and implement an intelligent DevSecOps platform, named WithOps, that unifies GitHub Actions workflow security analysis, automated maturity assessment, and AI-assisted threat modelling into a single integrated system. The platform was developed using a microservices architecture comprising nine independently deployable FastAPI services, a SvelteKit web frontend, and a Kong API Gateway, with inter-service communication via a Redis event bus and AI capabilities provided through multiple providers including Anthropic Claude, Meta Llama 3 via Groq, and locally hosted models through Ollama. A Retrieval-Augmented Generation system using Qdrant vector database was implemented to deliver context-aware conversational security insights. The completed platform detects 25 or more vulnerability patterns in GitHub Actions workflows, automatically computes DSOMM maturity scores across all five dimensions from Level 0 to Level 4, and performs multi-framework threat analysis using STRIDE, CIA triad, and LINDDUN methodologies from visual architecture diagrams. These findings demonstrate that an integrated, AI-powered platform can automate security assessment tasks that previously required 40 to 80 hours of manual evaluation, enabling organisations to continuously monitor and improve their DevSecOps posture without sacrificing development velocity.
