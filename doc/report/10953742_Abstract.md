## Abstract

Modern software development relies heavily on Continuous Integration and Continuous Deployment pipelines, with platforms such as GitHub Actions becoming the industry standard for workflow automation. However, research has shown that over 60% of these pipelines contain critical security misconfigurations — including unpinned dependencies, excessive permissions, and hardcoded credentials — making them attractive targets for supply chain attacks. Organisations simultaneously lack automated tools to assess their DevSecOps maturity against established frameworks such as the OWASP DevSecOps Maturity Model, where manual evaluations require 40 to 80 hours per assessment cycle. Threat modelling practices also remain disconnected from development workflows, leading a majority of teams to skip them entirely. This project aimed to design and implement an intelligent DevSecOps platform, named WithOps, that unifies GitHub Actions workflow security analysis, automated maturity assessment, and AI-assisted threat modelling into a single integrated system. The platform was developed using a microservices architecture comprising nine independently deployable FastAPI services, a SvelteKit 5 web frontend, and a Kong API Gateway, with inter-service communication via a Redis event bus. Artificial intelligence capabilities were delivered through a multi-provider strategy integrating Anthropic Claude, Meta Llama 3, and locally hosted models, augmented by a Retrieval-Augmented Generation system using a Qdrant vector database for context-aware conversational security insights. The completed platform successfully detected over 25 vulnerability patterns across GitHub Actions workflows, automated DevSecOps maturity scoring across all five OWASP DSOMM dimensions from Level 0 to Level 4, and performed multi-framework threat analysis using STRIDE, CIA triad, and LINDDUN methodologies through an interactive visual canvas with real-time multi-user collaboration. These findings demonstrate that a unified, AI-enhanced DevSecOps intelligence platform can substantially automate security assessment processes that previously demanded significant manual effort, enabling organisations to continuously monitor and improve their CI/CD security posture without sacrificing development velocity.

**Keywords:** CI/CD Pipeline Security, DevSecOps, OWASP DSOMM, Retrieval-Augmented Generation, Threat Modelling
